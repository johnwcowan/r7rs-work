Integer set procedures

The elements of an integer set are exact integers. The procedures for creating and manipulating integer sets are the same as those for SRFI 113 sets, except that set is replaced by iset in their names. However, set-comparator has no equivalent for integer sets, and in iset, iset-unfold, iset-map, and iset-copy the comparator argument is omitted.

Although sets subsume integer sets, providing them separately allows for increased implementation efficiency. There are two Chicken eggs that provide integer sets: iset, on which this SRFI is based, which uses bit vectors (and underlying them, bytevectors); and cis, which uses lists of integer intervals.

Procedures without SRFI-13 analogues

(make-iset-range low high)

Returns an iset containing the consecutive integers from low to high, inclusive.

(iset-min iset)

(iset-max iset)

Returns the smallest or largest integer in iset, or #f if there is none.

(iset-open-interval iset low high)

(iset-closed-interval iset low high)

(iset-open-closed-interval iset low high)

(iset-closed-open-interval iset low high)

Procedures that return a subset of iset contained in the interval from low to high. The interval may be open, closed, open below and closed above, or open above and closed below.

(iset-range= iset k)

(iset-range< iset k)

(iset-range<= iset k)

(iset-range> iset k)

(iset-range>= iset k)

Procedures that return an integer set containing the elements of iset that are equal to, less than, less than or equal to, greater than, or greater than or equal to k. Note that the result iset-range= contains at most one element.

(iset-delete-min! iset)

(iset-delete-max! iset)

Linear update procedures that return two values: the smallest or largest integer in iset or #f if there is none, and a set which contains all the elements of set except the integer being returned.
